# git worktree remove Requires --force for Claude Code Worktrees

`git worktree remove <path>` always fails for Claude Code worktrees because the `.claude/hooks/` directory is always present as **untracked** content.

## Error

```
fatal: '<path>' contains modified or untracked files, use --force to delete it
```

## Cause

Claude Code copies session hooks into each worktree at `.claude/hooks/`:
- `pre_tool_use.py`, `post_tool_use.py`, `stop.py`, `notification.py`
- `session_start.py`, `subagent_stop.py`, `pre_compact.py`, `user_prompt_submit.py`
- `utils/llm/`, `utils/tts/` subdirectories

These are harness artifacts — not committed to the repo, never tracked by git.

## Fix

Always use `--force`:

```bash
git worktree remove --force .claude/worktrees/<name>/
```

The hooks directory is safe to discard; it is regenerated by the harness on next use.
